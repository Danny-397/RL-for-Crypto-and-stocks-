THE FRAMEWORK

Engineered like a research project, not a toy.

Every layer is modular, individually testable, and built with the practices that separate a real RL system from a notebook demo.

⚡

PPO, from scratch

Clipped surrogate objective, GAE advantages, entropy bonus, orthogonal init and gradient clipping — implemented in PyTorch, not wrapped from a library.

🌐

Dual market environments

StockTradingEnv and CryptoTradingEnv share one BaseTradingEnv, so accounting, costs, and rewards never silently diverge.

🛡️

Leakage control

Chronological train / validation / test splits with scalers fit on training data only. The agent is scored on data it has never seen.

🎯

Risk-aware rewards

Return net of transaction costs, plus drawdown and turnover penalties — discouraging reckless, over-leveraged noise-trading.

📊

Honest backtests

Total return, annualised Sharpe, and max drawdown on a held-out split — the metrics a quant actually cares about.

🧩

Extensible by design

Add a market by subclassing one base class. Add an algorithm (SAC, DDPG) behind the same agent API. A recurrent (LSTM) PPO policy is fully implemented — flip one config flag.

HOW IT WORKS

From raw prices to a trained trader.

01

Ingest & engineer

Load OHLCV (or generate synthetic data), build returns, RSI, MACD, moving-average ratios and volatility features.

02

Observe

The agent sees a rolling window of features plus its own account state — position, cash, and normalised equity.

03

Act & learn

It outputs a target position in [-1, 1], the env applies costs & slippage, and PPO updates the policy from advantages.

04

Backtest

Run the trained agent deterministically on the held-out test split and score return, Sharpe, and drawdown.

REAL MARKETS · WALK-FORWARD · OUT-OF-SAMPLE

The honest numbers, on real market data.

No cherry-picking: each PPO agent trains on the older history of a basket of real tickers, then is backtested on the recent, held-out period of every ticker — against buy-&-hold.

📌 How to read this. These numbers are one favorable training seed — and on it the crypto agent looks like it beats buy-&-hold. Don't be fooled: re-run across many seeds and the edge evaporates (mean −17%, a confidence interval straddling zero, statistically indistinguishable from buy-&-hold; equities is significantly worse). There is no reliable, seed-robust edge on real markets — exactly what market efficiency predicts. The multi-seed significance study is what catches that false positive — and that discipline, not a fantasy return, is the contribution.
