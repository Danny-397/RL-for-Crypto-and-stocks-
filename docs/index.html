THE FRAMEWORK

Built from the algorithm up.

PPO, both environments, the data pipeline, and the evaluation are implemented here — no stable-baselines, no wrappers. Each layer is a separate, individually tested module.

PPO, from scratch

Clipped surrogate objective, GAE advantages, entropy bonus, orthogonal init and gradient clipping — implemented in PyTorch, not wrapped from a library.

Dual market environments

StockTradingEnv and CryptoTradingEnv share one BaseTradingEnv, so accounting, costs, and rewards never silently diverge.

Leakage control

Chronological train / validation / test splits with scalers fit on training data only. The agent is scored on data it has never seen.

Risk-aware rewards

Return net of transaction costs, plus drawdown and turnover penalties — discouraging reckless, over-leveraged noise-trading.

Honest backtests

Total return, annualised Sharpe, and max drawdown on a held-out split — the metrics a quant actually cares about.

Swap the policy or the market

The same PPO agent trades one asset or allocates across a whole basket (cross-sectional long/short). A recurrent (LSTM) policy is fully implemented, and new markets/algorithms drop in behind the same API.

Unified agent. Separate environments.

HOW IT WORKS

From raw prices to a trained trader.

01

Ingest & engineer

Load OHLCV (or generate synthetic data), build returns, RSI, MACD, moving-average ratios and volatility features.

02

Observe

The agent sees a rolling window of features plus its own account state — position, cash, and normalised equity.

03

Act & learn

It outputs a target position in [-1, 1], the env applies costs & slippage, and PPO updates the policy from advantages.

04

Backtest

Run the trained agent deterministically on the held-out test split and score return, Sharpe, and drawdown.

ABOUT

Why I built this.

I wanted to learn modern reinforcement learning by building it, not importing it. So I implemented PPO from scratch in PyTorch, wrote two custom market environments, engineered the whole data pipeline, and set out to answer an honest question: can one agent learn to trade two very different markets?

The hardest — and most valuable — part wasn't the algorithm; it was the evaluation. An early run made the crypto agent look like it crushed the market. Instead of shipping that, I built a multi-seed significance test — and it showed the "win" was mostly luck of the random seed, statistically tied with buy-&-hold. Learning to distrust a good-looking backtest and to prove it with statistics taught me more than any single training run.

What it demonstrates: I can take a non-trivial ML system end to end — algorithm, data, training, evaluation, and a live deployment — and hold my own results to a real scientific standard.
